# Plan: Scaffold & Infra (Phase 1)

## Summary
Stand up the foundation for the Workout Tracker PWA: a Next.js (App Router, TypeScript, Tailwind v4) project with shadcn/ui, Clerk authentication (v7) gating all app routes, and a Supabase Postgres database accessed through Drizzle ORM with the three-table workout schema pushed live. After this phase, a user can sign in, an empty protected home page renders for the authed user, and `drizzle-kit push` succeeds against Supabase.

## User Story
As a lifter, I want to sign into the app and have my account and an empty workout home screen, so that the app is ready to record workouts that belong only to me.

## Problem → Solution
**Current state:** Empty repository (`.git` + `.claude` only). No app, no auth, no database.
**Desired state:** A running Next.js app where Clerk sign-in works, every app route is protected, the home page renders only for authenticated users, and the Supabase schema (`workouts`, `workout_exercises`, `sets`) exists and is reachable via a typed Drizzle client.

### AUTH_MIDDLEWARE
```ts
// SOURCE: clerk.com/docs/reference/nextjs/clerk-middleware (Clerk v7)
// FILE: src/middleware.ts  (rename to src/proxy.ts ONLY if installed Next.js is 16+)
import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server'

const isPublicRoute = createRouteMatcher(['/sign-in(.*)', '/sign-up(.*)'])

export default clerkMiddleware(async (auth, req) => {
  if (!isPublicRoute(req)) {
    await auth.protect()
  }
})

export const config = {
  matcher: [
    // Skip Next internals + static files; always run for everything else
    '/((?!_next|[^?]*\\.(?:html?|css|js(?!on)|jpe?g|webp|png|gif|svg|ttf|woff2?|ico|csv|docx?|xlsx?|zip|webmanifest)).*)',
    '/(api|trpc)(.*)',
  ],
}
```

### SERVER_AUTH (get userId, redirect if absent)
```ts
// SOURCE: clerk.com/docs — auth() is async in v7, imported from /server
// FILE: src/lib/auth.ts  — centralizes the auth gate; reused by every server read/write
import { auth } from '@clerk/nextjs/server'
import { redirect } from 'next/navigation'

/** Returns the Clerk userId or redirects to sign-in. Use in Server Components/Actions. */
export async function requireUserId(): Promise<string> {
  const { userId } = await auth()
  if (!userId) redirect('/sign-in')
  return userId
}
```

### DRIZZLE_SCHEMA (adapted to Clerk text userId — NO users table)
```ts
// SOURCE: orm.drizzle.team/docs/column-types/pg + PRD schema (lines 115-118)
// FILE: src/db/schema.ts
import {
  pgTable, uuid, text, integer, real, boolean, timestamp, index,
} from 'drizzle-orm/pg-core'
import { relations } from 'drizzle-orm'

export const workouts = pgTable(
  'workouts',
  {
    id: uuid('id').defaultRandom().primaryKey(),
    userId: text('user_id').notNull(),            // Clerk user id, e.g. "user_2abc..."
    name: text('name'),
    startedAt: timestamp('started_at', { withTimezone: true }).defaultNow().notNull(),
    completedAt: timestamp('completed_at', { withTimezone: true }),
    createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
  },
  (t) => [index('workouts_user_id_idx').on(t.userId)],
)

export const workoutExercises = pgTable('workout_exercises', {
  id: uuid('id').defaultRandom().primaryKey(),
  workoutId: uuid('workout_id')
    .notNull()
    .references(() => workouts.id, { onDelete: 'cascade' }),
  wgerExerciseId: integer('wger_exercise_id').notNull(),
  name: text('name').notNull(),                   // denormalized from wger
  position: integer('position').notNull().default(0),
})

export const sets = pgTable('sets', {
  id: uuid('id').defaultRandom().primaryKey(),
  workoutExerciseId: uuid('workout_exercise_id')
    .notNull()
    .references(() => workoutExercises.id, { onDelete: 'cascade' }),
  setNumber: integer('set_number').notNull(),
  reps: integer('reps'),
  weight: real('weight'),                          // kg, raw number (POC: single unit)
  completed: boolean('completed').default(false),
})

export const workoutsRelations = relations(workouts, ({ many }) => ({
  exercises: many(workoutExercises),
}))
export const workoutExercisesRelations = relations(workoutExercises, ({ one, many }) => ({
  workout: one(workouts, { fields: [workoutExercises.workoutId], references: [workouts.id] }),
  sets: many(sets),
}))
export const setsRelations = relations(sets, ({ one }) => ({
  workoutExercise: one(workoutExercises, {
    fields: [sets.workoutExerciseId],
    references: [workoutExercises.id],
  }),
}))
```

## Notes
- **Version anchors (verified June 2026):** `@clerk/nextjs` ^7 (Core 3, async `auth()`), `drizzle-orm` ~0.45, `postgres` ~3.4, `drizzle-kit` ~0.31, Next.js 16 (Tailwind v4, Turbopack, React 19). If `create-next-app` later installs Next 17+, re-verify the middleware filename and Tailwind setup.
- **Schema deviation from generic Drizzle examples:** the common Drizzle/Supabase tutorial uses a `users` table with a uuid PK. This project deliberately does NOT — Clerk owns identity, so `workouts.user_id` is `text` holding the Clerk id and there is no `users` FK. This is the PRD's intentional "separate vendors, scope by userId in server code" decision.
- **Phase 2 (wger proxy) and Phase 3 (core loop) both depend on this phase** and can then proceed in parallel per the PRD; Phase 3 will add the userId-scoped data-access layer on top of the `db` client and `requireUserId()` established here.
