## Anti-Patterns Verdict

**LLM assessment: passes, with residual signature.** Committed dark palette, volt genuinely restrained, domain-deep UI (ghost placeholders, plate math, warm-up ramps) that no template has. Residual tells: uppercase tracked eyebrows as the *only* hierarchy device (~10 uses: "UP NEXT", "DONE TODAY", "WORKOUT IN PROGRESS"...), one uniform `rounded-2xl border bg-card` container everywhere, and three icon vocabularies mixed (hand-rolled SVGs + lucide + text glyphs `✕`/`↻`).

**Browser overlays**: not run — dev server was down; no live screenshots this round.

## Overall Impression

This is a designed product, not a generated one — the color discipline and lifter-native vocabulary are real. The biggest opportunity is emotional: the app detects PRs and computes session stats, then routes straight past them to the home screen at the exact moment of finish. Quiet utilitarianism reads as an accident of routing, not a position.

## What's Working

1. **Ghost placeholder system** (workout-logger.tsx:297-314): history wins, plan target fills in — exactly the right recall support mid-set, with documented fallback rationale.
2. **Draft/resume architecture surfaced honestly**: write-behind queue, offline hint, pulsing resume banner; the reassurance copy matches genuinely robust behavior underneath.
3. **Color discipline held in practice**: volt only on primary action + live/active state; muted-foreground tuned to ≥4.5:1. Rare to see the stated rule actually enforced.

## Priority Issues

- **[P1] Start-day lands on a read-only page** (start-day-button.tsx:33). Intent at "Start Bench Day" is to log; user lands on stats + an Edit button. Extra tap every session, inconsistent with Resume. Fix: route to `/workout/${id}/edit`; detail becomes the post-session view. — /impeccable polish
- **[P1] Unconfirmed, un-undoable exercise remove next to the plates button** (workout-logger.tsx:256-284). One thumb slip deletes logged sets and autosave persists it. Fix: inline "Removed — Undo" (5s) and spatial separation. — /impeccable harden
- **[P2] No end-of-session moment** (workout-logger.tsx:203 routes home on save). PR detection already exists on the detail page. Fix: navigate to detail post-save or show a summary sheet. — /impeccable delight
- **[P2] Ghosts can't be accepted.** Tapping the set-complete circle with empty fields should adopt ghost values — roughly halves logging cost for steady-state training. — /impeccable polish
- **[P2] Plate sheet dialog lacks focus management** (plate-sheet.tsx:92-108): no focus trap, no initial focus, no Escape, body scroll not locked. — /impeccable harden
- **[P3] Logger "Cancel" lies slightly** (workout/new/page.tsx:39): draft survives and reappears. Label "Close" or offer discard. — /impeccable clarify

## Persona Red Flags

**Casey (one-handed, interrupted)**: trash/plates adjacency at top-right of each exercise card — worst thumb zone on large phones. Search listbox (max-h-72) can cover the set the thumb was on. Otherwise genuinely well-built for interruption.

**Alex (power user)**: Start → detail → Edit hop costs a tap per session; no rest timer (session clock ticks, but between-sets is where lifters look); must retype ghosted values.

**Sam (accessibility)**: strong baseline (combobox ARIA, aria-pressed, role=status/alert, reduced-motion page transitions). Gaps: plate-sheet focus trap; `animate-ping` dot not reduced-motion gated (resume-session-card.tsx:25); set-complete aria-label never flips state; `<dd>` precedes `<dt>` in detail stat `<dl>` (workout/[id]/page.tsx:192-195).

## Minor Observations

- Home h1 "Workout Tracker" is the only unbranded thing in the app.
- `type="number"` inputs: desktop spinners + scroll-wheel mutation; prefer `type="text"` + `inputMode`.
- Stat values at `text-xl` undersell the "glanceable large numerals" mandate.
- Program builder placeholders assume deload knowledge.
- Buttons uppercase Inter vs headers uppercase Oswald — two uppercase voices.
- Delete uses native `window.confirm` — functional but tonally cold.

## Questions to Consider

1. Why does the app celebrate nothing? PRs are detected, then routed past at peak dopamine.
2. Is the eyebrow the design system? Strip the micro-labels and every card is identical — what's the second hierarchy device?
3. What is the workout detail page *for* during a live session? About-to-lift and reviewing share one read-only layout.
