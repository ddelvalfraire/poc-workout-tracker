## Priority Issues

- **[P0] Remove-set ✕ deletes a logged set instantly, no undo** (workout-logger.tsx:421-429). 36px target beside the weight input; autosave commits the loss in seconds. The exercise-level remove got an undo stack for exactly this reasoning — sets didn't. Fix: extend the undo strip to sets, or make ✕ two-tap.
- **[P1] Mid-session is emotionally flat**: no rest timer (the most-checked number between sets) and the session clock is text-sm muted (session-clock.tsx:29). Fix: display-font elapsed numeral + rest countdown auto-starting on set-complete.
- **[P1] Program builder has no draft persistence** — a 30-set program built on a phone dies with a backgrounded-tab kill; the logger solved this exact problem. Fix: reuse the draft-sync queue or sessionStorage.
- **[P2] Plate-sheet background not inert** — tab is trapped but a screen-reader virtual cursor walks behind the sheet. Fix: `inert` on the app root while open, or native `<dialog>`.
- **[P2] Destructive parity**: Delete equals Edit in width/weight on both action rows; unit-toggle error is an unannounced "!" with a `title` tooltip. Fix: demote Delete to ghost/text; visible retry text.
