## Priority Issues

- **[P1] Range-ghost partial adoption corrupts logs** (workout-logger.tsx `adoptable()` + fill): a plan ghost of "8–12" is rejected for reps but the weight still adopts — a completed set with weight and no reps. Fix: all-or-nothing adoption, or adopt repMin.
- **[P1] Volt discipline broken on program detail**: every day card renders a default-volt StartDayButton. Fix: volt only on the next day; others outline (demotion machinery exists).
- **[P2] Sub-44px frequent-tap targets**: set-remove X (36px, flush against the weight input) and history Repeat button. The set circle's `before:-inset-1.5` trick should be applied.
- **[P2] Icon semantics flip**: logger uses Trash2=exercise/X=set; builder uses Trash2=day/X=exercise+set. Same action should keep one glyph.
- **[P3] Peak-end copy + token drift**: "Save workout" → "Finish workout"; replace the hardcoded `text-amber-500` offline hint with a warning token (the only off-palette color left).
